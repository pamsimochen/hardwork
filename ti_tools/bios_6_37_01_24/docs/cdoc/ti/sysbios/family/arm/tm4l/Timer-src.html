    46    /*!
    47     *  ======== Timer ========
    48     *  Timer Peripherals Manager
    49     *
    50     *  This module manages the timer peripherals available on TM4L devices.
    51     *
    52     *  The Timer module supports the timer in 'one shot', 'continuous', and
    53     *  'dynamic' modes.
    54     *
    55     *  In 'one shot' mode, a timer function will "fire" (run) when the timer
    56     *  period expires. In 'one shot' mode this will only happen once.
    57     *
    58     *  In 'continuous' mode, the specified timer function will "fire" every
    59     *  time the period expires, throughout the lifetime of the program.
    60     *
    61     *  In 'dynamic' mode, the specified timer function will "fire" every
    62     *  time the period expires.  But the period of the timer can be changed
    63     *  dynamically, to correspond to the next tick interrupt needed from the
    64     *  timer.  This mode is used by the SYS/BIOS
    65     *  {@link ti.sysbios.knl.Clock Clock} module for implementing
    66     *  dynamic tick suppression, to reduce the number of interrupts from the
    67     *  timer to the minimum required for currently scheduled timeouts.
    68     *
    69     *  NOTE: In the current implementation on MSP430 Timers cannot be
    70     *  created dynamically at runtime by the application.  Timers must be
    71     *  created statically in the application configuration.

   142     *  <h3> Timer Mapping Tables </h3>
   143     *  <p>
   144     *  The Timer module allows the user to use and configure the various timers
   145     *  that exist on a particular device.  This is achieved by specifying a timer
   146     *  ID when statically creating the Timer instance.
   147     *  A convention is used to assign logical IDs to physical timers.  ID '0'
   148     *  corresponds to the first Timer_A peripheral (TA0).  ID '1' is assigned to
   149     *  the next Timer_A (TA1), until there are no more Timer_A peripherals.  Then
   150     *  the next ID is assigned to the first Timer_B peripheral (if present), and
   151     *  so on.
   152     *  These tables are provided to show which timers map to which timer IDs.
   153     *  </p>
   154     *  {@link ./doc-files/TimerTables.html Timer Mapping Tables}
   155     *  @p
   156     */
